# 快速入门

> 部署你的文档站点并完成首次更改。

完成本指南后，你将拥有一个已上线的文档站点，可以开始对其进行自定义和扩展。

<Info>
  **前提条件**：在开始之前，请先[创建账户](https://mintlify.com/start)并完成引导流程。
</Info>

<div id="getting-started">
  ## 快速开始
</div>

完成入门引导后，你的文档站点会自动部署到一个符合以下格式的唯一 URL：

```
https://<你的项目名称>.mintlify.app
```

在你的[控制台](https://dashboard.mintlify.com/)的概览（Overview）页面中找到你的 URL。

<div id="install-the-github-app">
  ### 安装 GitHub 应用
</div>

Mintlify 提供了一个 GitHub 应用，你在将更改推送到仓库时，它会自动触发部署。

按照新手引导清单或仪表盘中的指引来安装 GitHub 应用。

1. 在 Mintlify 仪表盘中前往 **Settings**。
2. 在侧边栏中选择 **GitHub App**。
3. 选择 **Install GitHub App**。这会在新标签页中打开 GitHub 应用的安装页面。
4. 选择你希望安装该应用的组织或用户账号。
5. 选择你想要连接的仓库。

<div id="authorize-your-github-account">
  ### 授权你的 GitHub 账户
</div>

1. 在 Mintlify 控制台前往 **Settings**。
2. 在侧边栏选择 **My Profile**。
3. 选择 **Authorize GitHub account**。会在新标签页中打开 GitHub 授权页面。

<Info>
  根据你组织的设置，你的 GitHub 组织管理员可能需要为你的账户进行授权。
</Info>

<div id="code-based-workflow">
  ## 基于代码的工作流
</div>

基于代码的工作流可与您现有的开发环境和 Git 仓库集成。该工作流最适合希望将文档与代码并行管理的技术团队。

<div id="install-the-cli">
  ### 安装 CLI
</div>

<Info>
  **前置条件**：CLI 需要 [Node.js](https://nodejs.org/en) v20.17.0 至 v24 之间的版本（含 v20.17.0），推荐使用 LTS 版本。
</Info>

要在本地编辑你的文档，请在终端中运行以下命令安装名为 [mint](https://www.npmjs.com/package/mint) 的命令行工具（CLI）：

<CodeGroup>
  ```bash npm theme={null}
  npm i -g mint
  ```

  ```bash pnpm theme={null}
  pnpm add -g mint
  ```
</CodeGroup>

<div id="create-a-new-project">
  ### 创建新项目
</div>

运行 `mint new` 以创建一个新的文档项目。关于该命令及其可用选项的详细说明，请参阅 [命令行界面（CLI）安装指南](/zh/installation#create-a-new-project)。

<div id="edit-the-documentation">
  ### 编辑文档
</div>

完成项目设置后，你就可以开始编辑文档文件了。例如，更新介绍页面的标题：

1. 打开你的文档存储库。
2. 打开 `index.mdx`，找到文件顶部：

```mdx index.mdx theme={null}
---
title: "简介"
description: "这是文档的介绍"
---
```

3. 将 `title` 字段更新为 `"Hello World"`。

```mdx index.mdx {2} theme={null}
---
title: "Hello World"
description: "这是文档的介绍"
---
```

<div id="preview-the-changes">
  ### 预览更改
</div>

要在本地预览这些更改，请运行以下命令：

```bash  theme={null}
mint dev
```

你可以在 `localhost:3000` 查看预览。

<div id="push-the-changes">
  ### 推送更改
</div>

当你准备好发布更改时，将它们推送到你的代码仓库。

Mintlify 会自动检测这些更改，构建你的文档，并将更新部署到你的站点。你可以在 GitHub 仓库的提交记录或在 [仪表盘](https://dashboard.mintlify.com) 中监控部署状态。

部署完成后，你的最新更新将可以通过 `<your-project-name>.mintlify.app` 访问。

<Card title="前往添加自定义域名" icon="arrow-down" horizontal href="#adding-a-custom-domain">
  你也可以选择跳过网页编辑器的流程，直接前往添加自定义域名。
</Card>

<div id="web-editor-workflow">
  ## Web editor 工作流
</div>

Web editor 工作流提供 WYSIWYG（所见即所得）界面，用于创建和编辑文档。该工作流最适合希望直接在网页浏览器中工作且无需额外本地开发工具的用户。

<div id="access-the-web-editor">
  ### 访问网页编辑器
</div>

1. 登录你的[控制台](https://dashboard.mintlify.com)。
2. 在左侧边栏选择 **Editor**。

<Info>
  如果你还没有安装 GitHub App，打开网页编辑器时系统会提示你安装该应用。
</Info>

<div id="troubleshooting">
  ## 故障排查
</div>

如果你在配置过程中遇到问题，请查看以下常见解决方案：

<AccordionGroup>
  <Accordion title="本地预览无法运行">
    请确保你已安装 Node.js v20.17.0 或更高版本，并且在包含 `docs.json` 文件的目录中运行 `mint dev` 命令。
  </Accordion>

  <Accordion title="更改未在线上站点生效">
    部署可能需要几分钟时间。请检查你的 GitHub Actions（适用于基于代码的工作流）或 Mintlify 控制台中的部署日志，以确保没有构建错误。
  </Accordion>

  <Accordion title="自定义域名无法连接">
    请确认你的 DNS 记录配置正确，并预留足够的 DNS 传播时间（最长可达 48 小时）。你可以使用 [DNSChecker](https://dnschecker.org) 等工具来验证你的 CNAME 记录。
  </Accordion>
</AccordionGroup>
